# 5Q診断 - Manus指示書

## プロジェクト概要
5つの知能タイプ（IQ、EQ、AQ、SQ、XQ）を診断するWebアプリケーション。

## 基本仕様

### 設問設計
- **総問数**: 50問（各診断10問）
- **出題方式**: 5科目がシャッフルされてランダムに出題
- **選択肢設計**: 社会的望ましさバイアスを排除
  - どの選択肢が高得点かが一目で分からない
  - すべての選択肢に合理性・強み・弱点を含める
  - 高得点選択肢にも副作用を含める

### 回答方法についての注意
- 診断にかかる時間：全50問、約10～15分
- 回答方法：各質問に対して、あなたの最初の反応を選んでください

### 結果表示順序
1. 5Qスコア一覧
2. 統合タイプ
3. 参考人物イメージ
4. 各診断の個別要約
5. **総合解説** ← 新規追加
6. 強み
7. ボトルネック
8. 成長提案
9. もう一度診断するボタン

## 追加指示：診断結果に総合解説を追加する

### 目的
ユーザーが点数やタイプを見て終わるのではなく、以下を理解できるようにする：
- 自分はどの能力で成果を出しやすいのか
- どの能力が補助的に働いているのか
- どこがボトルネックになりやすいのか
- 今後どこを伸ばすとよいのか

### 総合解説に含める内容

#### 1. 能力バランスの解説
5つのスコアを比較し、どの能力が中心になっているかを説明する。

**例**：
- IQよりもAQ・SQ・XQが高い場合：
  「あなたは単純な思考速度よりも、経験を構造化し、組織や状況を読みながら成果につなげるタイプです。」

- IQが高くEQが低い場合：
  「論理判断は強い一方で、相手の感情処理や納得形成を後回しにしやすい傾向があります。」

- EQ・SQが高い場合：
  「人の感情だけでなく、集団内の関係性や空気を読んで動けるタイプです。」

- AQ・XQが高い場合：
  「失敗や逆境を経験値に変え、次の行動に再利用する力が強いタイプです。」

#### 2. 成果の出し方
その人がどのように成果を出しやすいかを説明する。

**パターン例**：
- 思考で勝つ
- 人を動かして勝つ
- 逆境を突破して勝つ
- 経験を積み上げて勝つ
- 構造や仕組みを作って勝つ
- バランス型で安定して勝つ

#### 3. つまずきやすい場面
高い能力の副作用や、低い能力とのギャップを説明する。

**例**：
- AQとXQが高く、EQが相対的に低い場合：
  「自分は失敗から学べるため、他人にも同じ速度の改善を求めやすい傾向があります。」

- SQが高くIQが低めの場合：
  「人や場は読めますが、複雑な構造設計や数値分析では補助ツールを使うと安定します。」

- IQが高くAQが低めの場合：
  「考える力は強い一方で、失敗後の立て直しや長期継続で消耗しやすい可能性があります。」

#### 4. 成長提案
総合的に見て、最も伸ばすべき能力を1～2個提示する。

**選び方**：
- 最低スコアの能力
- 最高スコアを活かすために必要な補助能力
- スコア差が大きい能力

**例**：
「あなたの場合、次に伸ばすべきはEQです。理由は、AQ・XQが高いため、自分の改善速度を他人にも求めやすくなるからです。EQを伸ばすことで、成果を出す力が人を動かす力に変わります。」

### 総合解説の文量
- 300～600文字程度
- 長すぎない
- 読み物として納得感がある
- 点数の羅列ではなく、意味づけをする

### 表示テンプレート

```
総合解説

あなたの5Qバランスを見ると、中心になっている能力は「XQ」と「AQ」です。
これは、単純に考える力だけで成果を出すタイプではなく、経験から学び、困難を突破しながら成果を積み上げるタイプだといえます。

一方で、EQが相対的に低い場合、他人の感情整理や納得形成を待つ場面でストレスを感じやすいかもしれません。
あなたにとって自然な改善速度が、周囲にとっては速すぎる場合があります。

今後さらに成果を伸ばすには、XQで得た学びを自分だけで使うのではなく、他人にも使える形に翻訳することが重要です。
そのためには、EQやSQを補助的に伸ばすことで、個人の成果が組織の成果に変わりやすくなります。
```

### 条件分岐ロジック

総合解説は、以下のようにスコア傾向によって文言を変える。

#### パターン1：IQ突出型
**条件**: IQが5項目中最高、かつ75点以上

**解説**:
- 思考力・論理力で成果を出すタイプ
- 注意点：考えすぎ・説明過多・行動遅れ
- 成長提案：AQやXQを伸ばして、思考を行動に変える

#### パターン2：EQ突出型
**条件**: EQが最高、かつ75点以上

**解説**:
- 感情理解・関係形成で成果を出すタイプ
- 注意点：周囲に影響されすぎること
- 成長提案：IQやAQを伸ばして、感情を戦略に変える

#### パターン3：AQ突出型
**条件**: AQが最高、かつ75点以上

**解説**:
- 逆境突破で成果を出すタイプ
- 注意点：無理を続けやすいこと
- 成長提案：EQやSQを伸ばして、個人の突破を組織の突破に変える

#### パターン4：SQ突出型
**条件**: SQが最高、かつ75点以上

**解説**:
- 人間関係・組織構造・空気を読んで成果を出すタイプ
- 注意点：読みすぎ・政治的になりすぎること
- 成長提案：IQやXQを伸ばして、読みを設計に変える

#### パターン5：XQ突出型
**条件**: XQが最高、かつ75点以上

**解説**:
- 経験学習・抽象化・転用で成果を出すタイプ
- 注意点：飽きやすさや他人への速度要求
- 成長提案：EQやSQを伸ばして、学びを共有に変える

#### パターン6：AQ・SQ・XQ高水準型（変革リーダー型）
**条件**: AQ、SQ、XQがすべて75点以上

**解説**:
- 変革リーダー型
- 経験を構造化し、人や組織を読み、逆境でも仕組みを変えて成果を出すタイプ
- 注意点：人を構造変数として扱いすぎること
- 成長提案：EQを伸ばして、設計を共感に変える

#### パターン7：IQ・AQ・XQ高水準型（戦略実行型）
**条件**: IQ、AQ、XQがすべて75点以上

**解説**:
- 戦略実行型
- 考える、試す、改善するの回転が速いタイプ
- 注意点：感情面や関係調整を軽く見やすいこと
- 成長提案：EQやSQを伸ばして、実行を協働に変える

#### パターン8：EQ・SQ高水準型（人間理解型）
**条件**: EQ、SQが75点以上

**解説**:
- 人間理解型・調整支援型
- 個人の感情と集団の関係性の両方を扱えるタイプ
- 注意点：自分の意思決定が遅れること
- 成長提案：IQやAQを伸ばして、理解を判断に変える

#### パターン9：全体バランス型
**条件**: 全項目が60～74点、または極端な突出がない

**解説**:
- 大きな弱点は少なく、環境に応じて能力を使い分けるタイプ
- 注意点：突出した武器が見えにくいこと
- 成長提案：1つの能力を意識的に伸ばして、専門性を作る

#### パターン10：全体低め型
**条件**: すべて60点未満

**解説**:
- 現時点では特定能力で押し切るより、環境選びと小さな習慣化で伸ばすタイプ
- **重要**: 否定的に表現しない
- 「未発達」ではなく「これから伸ばしやすい状態」と表現する
- 成長提案：得意な環境を見つけて、1つの能力から伸ばす

### 実装データ例

`results.js` に `integratedExplanations` を追加してください。

```javascript
const integratedExplanations = {
    transformationLeader: {
        title: "総合解説：変革リーダー型",
        condition: "aq >= 75 && sq >= 75 && xq >= 75",
        text: "あなたの5Qバランスを見ると、AQ・SQ・XQが強く出ています。これは、困難に耐えるだけでなく、状況や組織の構造を読み、経験から学んだことを使って仕組みごと変えていくタイプです。単純な個人能力よりも、構造を設計して成果を再現する力が中心になります。一方で、自分が改善できることを他人にも求めすぎると、周囲がついてこられない場合があります。今後は、正しい設計だけでなく、相手が動ける設計に変換することが重要です。"
    },
    // 他のパターンも同様に追加

```

**総合解説は、単なる点数説明ではなく、ユーザーが読んで「自分の成果パターンがわかった」と感じる文章にしてください。**

## 追加指示：心理学テスト型に設問を全面改修する

### 目的

現在の設問は、回答者が「どれが高得点か」を推測しやすいため、設問を心理学テスト型に作り直す。

- **直接的な自己評価ではなく**、自然な行動選択からIQ / EQ / AQ / SQ / XQの傾向を測る
- 回答者が何を測られているか分かりにくいが、結果を見ると「確かに自分に近い」と感じる診断にする

### 設問構成比率

各診断20問のうち、以下の割合で作成する：

| タイプ | 比率 | 問数 |
|--------|------|------|
| ストーリー型 | 40% | 8問 |
| トレードオフ型 | 30% | 6問 |
| 強制選択型 | 20% | 4問 |
| 直接質問型 | 10% | 2問 |

**40問プールの場合**：
- ストーリー型：16問
- トレードオフ型：12問
- 強制選択型：8問
- 直接質問型：4問

### 設問タイプ定義

#### 1. ストーリー型（40%）

具体的な場面を提示し、その中で最も近い行動を選ばせる。

**例**：
```
あなたは新しい部署の責任者になった。
最初の1週間で、次のことに気づいた。

* ベテラン社員は優秀
* 若手は萎縮している
* 売上は悪くない
* 離職率が高い

最初に行うこととして近いのは？

A：個別面談を行う
B：数字と業務フローを確認する
C：会議を観察する
D：評価制度や役割配置を確認する
```

#### 2. トレードオフ型（30%）

どれも合理的だが、何を優先するかで傾向が出る設問。

**例**：
```
重要なプロジェクトが遅れている。
最も優先するものは？

A：品質を守る
B：納期を守る
C：関係者の納得を取る
D：次回以降の再発防止を設計する
```

#### 3. 強制選択型（20%）

どちらも正しい価値観を並べて、近い方を選ばせる。
4択にする場合は、2軸×2段階で構成する。

**例**：
```
どちらに近いですか？

A：納得してから動く方が多い
B：動きながら納得していく方が多い
C：人を巻き込む前に整理する方が多い
D：人と話しながら整理する方が多い
```

#### 4. 直接質問型（10%）

自己認識を問う。ただし、全体の10%までに抑える。

**例**：
```
自分が最も得意だと感じるのは？

A：論理的に整理すること
B：人の気持ちを読むこと
C：困難でも立て直すこと
D：経験を次に活かすこと
```

### 最重要ルール：禁止する設問

以下のような直接的で分かりやすい設問は禁止。社会的望ましさバイアスが強く出る。

- 失敗したら原因を分析しますか？
- あなたは逆境に強いですか？
- 人の気持ちを読めますか？
- 経験から学べますか？
- 論理的に考えますか？

### 選択肢設計ルール

各選択肢には必ず合理性を持たせる。

**悪い例**：
```
A：怒る
B：落ち込む
C：考える
D：仕組みにする
```

**良い例**：
```
A：まず現場の混乱を抑える
B：関係者の認識差を確認する
C：原因が個人か構造かを切り分ける
D：短期対応と再発防止を分けて考える
```

### 高得点を隠すルール

1. 高得点選択肢だけを立派に書かない
2. 低得点選択肢も自然で合理的に見せる
3. 各選択肢の文章量を近づける
4. 高得点選択肢にも副作用を含める
5. 低得点選択肢にも強みを含める
6. 「正解っぽい言葉」を連発しない
7. A/B/C/Dにスコア傾向を固定しない

### 逆転設問（20～30%）

全体の20～30%は逆転設問にする。

**逆転設問とは**：一見よさそうな選択肢が必ずしも高得点ではない問題。

**例**：
```
会議で反対意見が出なかった。

A：合意とみなし、決定を進める
B：発言していない人の反応を見る
C：もう一度全員に意見を求める
D：決定後の不満が出る前提で進める
```

この場合、「全員に意見を求める」は一見よいが、必ずしも最高点ではない。
状況によっては冗長化・意思決定遅れになる。

### 社会的望ましさバイアス対策

#### IQ
- 「難しいことを考える」「論理的に整理する」が常に高得点にならないようにする
- 実務で必要な判断、仮説、優先順位、情報圧縮を見る

#### EQ
- 「優しくする」「共感する」が常に高得点にならないようにする
- 共感だけでなく、感情を扱って行動に変える力を見る

#### AQ
- 「耐える」「諦めない」が常に高得点にならないようにする
- 根性論ではなく、回復・適応・再設計を見る

#### SQ
- 「空気を読む」「人に合わせる」が常に高得点にならないようにする
- 集団構造・利害・役割・影響力を見る

#### XQ
- 「反省する」「学ぶ」が常に高得点にならないようにする
- 経験を抽象化し、転用し、再現できるかを見る

### questions.js の追加データ構造

各設問に `questionType` を追加する。

```javascript
questionType: "story" | "tradeoff" | "forced" | "direct"
```

また、必要に応じて `trap` を付ける（逆転設問の場合）。

**例**：
```javascript
{
    id: "aq_story_001",
    category: "adaptation",
    questionType: "story",
    trap: true,
    text: "重要案件が急に白紙になった。翌日に最も近い行動は？",
    choices: [
        {
            text: "関係者の不安を抑え、まず通常運転に戻す",
            level: 2
        },
        {
            text: "白紙になった理由と影響範囲を確認する",
            level: 4
        },
        {
            text: "気持ちを切り替え、別案件へ集中する",
            level: 1
        },
        {
            text: "使える資料や関係性を次案件に転用する",
            level: 3
        }
    ]
}
```

### 出題ロジックの追加条件

20問出題時は以下を満たす：

- 各カテゴリから4問ずつ
- questionType比率を可能な限り守る
  - story：8問
  - tradeoff：6問
  - forced：4問
  - direct：2問
- trap設問を20～30%含める
- 出題後に順番をシャッフルする
- **選択肢も毎問シャッフルする**（重要）

### 設問作成例

以下のような方向で、既存設問を作り直す：

#### AQ ストーリー型
```
あなたが責任者の施策で、想定より成果が出なかった。
翌週の会議で最初に話す内容として近いのは？

A：想定と結果の差分を整理して共有する
B：関係者の努力を認め、次の挑戦につなげる
C：外部要因と内部要因を分けて確認する
D：今回の学びを次回の判断基準に変える
```

#### EQ トレードオフ型
```
相手が明らかに落ち込んでいる。
ただし、締切も迫っている。

A：まず気持ちを聞く時間を作る
B：最低限の作業範囲を一緒に決める
C：本人が話し出すまで距離を置く
D：周囲に負担が偏らないよう調整する
```

#### SQ ストーリー型
```
会議では全員が賛成しているが、終了後に一部の人が不満を漏らしている。

A：次回から会議中に反対意見を出しやすくする
B：不満を言っている人の立場や利害を確認する
C：決定事項は維持し、運用で微調整する
D：発言しなかった理由を個別に確認する
```

#### XQ トレードオフ型
```
別業界の成功事例を見つけた。
最初に考えることは？

A：自分の業界でもそのまま使えるか
B：成功の前提条件は何か
C：どの部分なら小さく試せるか
D：過去の経験と似ている点は何か
```

#### IQ 強制選択型
```
情報が多すぎて判断しづらい時、近いのは？

A：重要そうな情報から順に読む
B：まず判断に不要な情報を捨てる
C：全体像をつかんでから細部を見る
D：仮説を置いて必要情報だけ集める
```

### 結果への反映

設問が心理学テスト型になるため、結果画面では以下も表示する：

- あなたが重視しやすい判断軸
- 無意識に選びやすい行動パターン
- 高く出た能力の使い方
- 低く出た能力の補助方法

### 最終品質基準

設問を見たユーザーが、
**「これはどれが高得点なのか分からない」**
と感じること。

結果を見たユーザーが、
**「でも確かに自分の傾向が出ている」**
と感じること。

この2つを最重要基準にする。
